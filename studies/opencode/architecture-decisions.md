# Architecture Decisions

This file records important architecture decisions for the OpenCode study and the initial AI-LAB setup.

ADRs are only for decisions with meaningful architectural consequences. Do not use ADRs for routine notes.

## ADR-0000

Title:
Birth of AI-LAB as a lightweight laboratory

Date:
2026-06-27

State:
accepted

Context:
AI-LAB is starting as a long-term engineering laboratory for studying AI architectures applied to software development. OpenCode is the first case study, but the lab must remain lightweight enough to start real investigation immediately.

Decision:
Adopt a minimal viable lab based on:

- Questions.
- Claims.
- Evidence.
- Experiments.
- Runs.
- Provisional specs.
- ADRs only when necessary.
- Patterns only when earned.

Deferred intentionally:

- Real knowledge graph.
- Formal ontology.
- Formal governance.
- Dashboards.
- Separate benchmark system.
- Publications.
- Automation.

Consequences:
The lab can begin OpenCode research immediately with low maintenance cost while preserving evidence, reproducibility, and explicit uncertainty.
