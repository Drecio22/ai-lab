## Key divergence from static trace

The static trace predicted `session.next.*` events (session.next.text.delta, session.next.text.ended, etc.). In the actual runtime, events were published through the v1 event path:
- `message.part.delta` instead of `session.next.text.delta`
- `message.updated` instead of `session.next.*`
- `message.part.updated` instead of `session.next.*`

This is because the test configuration uses `experimentalEventSystem: true` which routes through the v1 path. The v2 `session.next.*` path is an alternative event system that's not the default.

The actual event system used in this commit is the v1 path, which is what the client-side stores consume. The static trace should be updated to reflect the v1 event type names for accuracy.
